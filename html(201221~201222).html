필자는 팀 버너스리가 처음으로 창조한 웹의 최초의 모습을 원시웹이라 부른다.
WEB1은 바로 이 원시웹을 다루는 수업

인터넷이 동작하기 위해서는 컴퓨터가 최소 2대 있어야 한다.
Web browser와 Web server
Browser는 정보를 요청하고 Server는 정보 요청에 반응한다.
Client와 Server
폰 노이만: 수학은 이해하는 것이 아니라 익숙해지는 것이다.

웹서버에 연결하는 두 가지 방법
1. 컴퓨터에 직접 웹서버 설치 - 어려운 길이지만 많은 것을 알 수 있는 길
2. 웹서버 제공해주는 웹호스팅 업체 이용 - 쉬운 길이지만 많은 것이 감춰진 길

수많은 웹호스팅 업체들 중 하나 github의 pages 기능

웹서버 제품군에는 Apache, IIS, Nginx 등이 있다.
내 컴퓨터의 서버가 할당하고 있는 웹페이지 = localhost
http://localhost/index.html
http://127.0.0.1/index.html
domain name: localhost와 같이 의미가 있는 이름
ip address: 127.0.0.1과 같이 번호로 되어 있는 것
hydocs: hypertext documents, 웹페이지가 저장된 디렉토리
HTTP: HyperText Transfer Protocol, 웹페이지 전송을 위한 통신 규약

웹서버와 같은 기기가 아닌 웹클라이언트를 통해 서버에 접속하기
127.0.0.1: 일종의 재귀대명사
ipconfig를 통해 나타난 내 컴퓨터의 IP 주소: 일종의 고유명사
그런데 ip주소를 통한 직접접속의 조건: 두 기기(클라이언트와 서버)가 같은 네트워크로 연결되어 있어야 한다.

아쉽게도 우리 수업은 도메인 이름을 사용해서 접속하는 방법, 다른 네트워크에서의 웹서버 접속 방법 등은 다루지 못했다.
